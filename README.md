# ASR Project - Hindi-English Code-Switched Automatic Speech Recognition

This project provides tools for fine-tuning and evaluating Wav2Vec2 models for Automatic Speech Recognition (ASR), specifically targeting Hindi-English code-switched audio using the CS-FLEURS dataset and evaluating against both synthetic-style and real-world test sets.

## Project Overview

This project focuses on:

- **Fine-tuning** Wav2Vec2 models for Hindi-English code-switched ASR tasks
- **Evaluating** models on multiple datasets
- **Building custom vocabularies** from training data
- **Computing metrics** like Word Error Rate (WER) and Character Error Rate (CER)
- **Comparing synthetic-style and real-world evaluation performance**

The project uses the Hugging Face `transformers` library and supports both CPU and CUDA GPU training/inference.

## Real-World Test Data Setup (OpenSLR104)

The `test/` folder is **not generated by the training script**. It must be created manually by downloading the Hindi-English test data from OpenSLR 104 and extracting it into the project.

- URL: `https://www.openslr.org/104/`
- File to download: `Hindi-English_test.tar.gz`

This dataset is used as the **real-world evaluation set** for the Hindi-English ASR model.

### Download and extract

From the project root:

```bash
cd /workspace/projects/asr_project_src
mkdir -p test_download
cd test_download

wget https://www.openslr.org/resources/104/Hindi-English_test.tar.gz
tar -xzf Hindi-English_test.tar.gz
```

### Organize into the expected structure

After extraction, place the audio files and Kaldi transcript files into this structure:

```text
test/
├── transcripts/
│   ├── wav.scp
│   ├── segments
│   ├── text
│   ├── utt2spk
│   ├── spk2utt
│   └── spkr_list
└── *.wav
```

### What these files mean

- `wav.scp`: maps recording IDs to WAV filenames
- `segments`: maps utterance IDs to `(recording_id, start_time, end_time)`
- `text`: reference transcript for each utterance
- `utt2spk`, `spk2utt`, `spkr_list`: speaker metadata

The Hindi-English evaluation script uses:

- `wav.scp`
- `segments`
- `text`

to cut utterance-level segments from the full WAV files and compute WER/CER on the real-world dataset.

## Scripts Overview

### 1. `ahmer/finetune_hin-eng.py` - Hindi-English Fine-tuning

Fine-tunes a Wav2Vec2 model on CS-FLEURS Hindi-English code-switched data.

**Features:**
- Downloads CS-FLEURS dataset automatically
- Builds a custom vocabulary from text transcripts
- Splits data into train/eval
- Fine-tunes a Wav2Vec2 model
- Saves checkpoints and final model into `runs/`

**Usage from root directory:**
```bash
python ahmer/finetune_hin-eng.py \
  --output_dir runs/w2v2_csfleurs_hin_eng \
  --num_train_epochs 50 \
  --batch_size 4 \
  --learning_rate 3e-4 \
  --device cuda
```

**Important arguments:**
- `--base_model`: base Hugging Face model
- `--output_dir`: where to save model/checkpoints
- `--num_train_epochs`: number of epochs
- `--batch_size`: batch size
- `--grad_accum`: gradient accumulation steps
- `--learning_rate`: learning rate
- `--max_samples`: limit training samples
- `--device`: `cuda` or `cpu`

---

### 2. `ahmer/evaluate_hin-eng.py` - Hindi-English Evaluation

Evaluates a trained model on:

1. **CS-FLEURS `xtts_test1` Hindi-English** — synthetic / cleaner benchmark-style test data
2. **OpenSLR104 Hindi-English test set** — real-world spoken tutorial audio downloaded from OpenSLR and extracted locally into the `test/` folder

**Features:**
- Loads a fine-tuned Hindi-English model from `runs/`
- Evaluates on filtered CS-FLEURS Hindi-English test data
- Evaluates on real-world OpenSLR104 Hindi-English test data
- Handles Kaldi-format metadata (`wav.scp`, `segments`, `text`)
- Computes **WER** and **CER**
- Saves final results to JSON

**Usage from root directory:**
```bash
python ahmer/evaluate_hin-eng.py \
  --model_path runs/w2v2_csfleurs_hin_eng \
  --device cuda \
  --lang_pair hin-eng \
  --csfleurs_subset xtts_test1 \
  --real_wav_root test \
  --real_wav_scp test/transcripts/wav.scp \
  --real_segments test/transcripts/segments \
  --real_text test/transcripts/text \
  --save_json ahmer/eval_results.json
```

**Arguments:**
- `--model_path`: path to fine-tuned model directory
- `--device`: `cuda` or `cpu`
- `--lang_pair`: language pair, default `hin-eng`
- `--csfleurs_subset`: currently `xtts_test1`
- `--max_samples_csfleurs`: optional sample cap for CS-FLEURS
- `--max_samples_real`: optional sample cap for OpenSLR104
- `--real_wav_root`: directory containing extracted test WAV files
- `--real_wav_scp`: path to Kaldi `wav.scp`
- `--real_segments`: path to Kaldi `segments`
- `--real_text`: path to Kaldi `text`
- `--save_json`: optional output JSON path

## Evaluation Results

The evaluation results compare the same fine-tuned model on two datasets:

1. **CS-FLEURS `xtts_test1` Hindi-English** — synthetic / cleaner benchmark-style test data
2. **OpenSLR104 Hindi-English test set** — real-world spoken tutorial audio downloaded from OpenSLR and extracted locally into the `test/` folder

Example result file:

```json
{
  "model_path": "/workspace/projects/asr_project_src/runs/w2v2_csfleurs_hin_eng",
  "csfleurs": {
    "dataset": "csfleurs_xtts_test1",
    "lang_pair": "hin-eng",
    "num_scored": 418,
    "num_skipped": 0,
    "wer": 0.4034866828087167,
    "cer": 0.18972339256274043
  },
  "openslr104_real": {
    "dataset": "openslr104_hin_eng_real_test",
    "num_scored": 3136,
    "num_skipped": 0,
    "wer": 0.7691366887346787,
    "cer": 0.5916973527716175
  }
}
```

### Metric explanation

- **WER (Word Error Rate)**: percentage of words predicted incorrectly
- **CER (Character Error Rate)**: percentage of characters predicted incorrectly
- **num_scored**: number of successfully evaluated items
- **num_skipped**: number of items skipped due to loading or processing issues

### Interpretation of the current result

- **CS-FLEURS (`xtts_test1`)**
  - WER: **40.35%**
  - CER: **18.97%**
  - Samples scored: **418**
- **OpenSLR104 real-world**
  - WER: **76.91%**
  - CER: **59.17%**
  - Segments scored: **3136**

This indicates the model performs substantially better on the cleaner / synthetic-style CS-FLEURS evaluation set than on the real-world Hindi-English spoken-tutorial data from OpenSLR104.

## Model Information

### Main trained model: `runs/w2v2_csfleurs_hin_eng/`

This folder contains the fine-tuned Hindi-English Wav2Vec2 model and all files needed for inference or evaluation.

### What is stored in `runs/`

The `runs/` directory stores:

- final trained model folders
- intermediate checkpoints
- vocabulary files
- processor/tokenizer configs
- training arguments
- evaluation outputs such as saved JSON results
- additional experiments and debug runs

### Example contents of `runs/w2v2_csfleurs_hin_eng/`

- `model.safetensors` — fine-tuned model weights
- `config.json` — model configuration
- `vocab.json` — vocabulary built from training data
- `processor_config.json` / `preprocessor_config.json` — processor settings
- `tokenizer_config.json` / `special_tokens_map.json` — tokenizer metadata
- `training_args.bin` — saved Hugging Face training arguments
- `checkpoint-*` — intermediate training checkpoints

### Base model

- `facebook/wav2vec2-xls-r-300m`

### Fine-tuning data

- CS-FLEURS Hindi-English code-switched training data

### Example inference usage

```python
from transformers import Wav2Vec2Processor, Wav2Vec2ForCTC
import librosa
import torch

processor = Wav2Vec2Processor.from_pretrained("runs/w2v2_csfleurs_hin_eng")
model = Wav2Vec2ForCTC.from_pretrained("runs/w2v2_csfleurs_hin_eng")

audio, sr = librosa.load("path/to/audio.wav", sr=16000)

inputs = processor(audio, sampling_rate=16000, return_tensors="pt")
with torch.no_grad():
    logits = model(inputs.input_values).logits

predicted_ids = torch.argmax(logits, dim=-1)
transcription = processor.batch_decode(predicted_ids)[0]
print(transcription)
```

## Troubleshooting

### CUDA out of memory
Reduce batch size, for example:
```bash
--batch_size 2
```

### Audio files not found
Check:
- `test/` contains WAV files
- `test/transcripts/` contains `wav.scp`, `segments`, and `text`
- paths passed to the evaluation script are correct

### Missing dependencies
Run:
```bash
pip install -r requirements.txt
```

### CS-FLEURS test download is too large
If evaluating CS-FLEURS, make sure the evaluation script downloads metadata first, filters `hin-eng`, then downloads only the matching audio files.

### Dataset download issues
Check internet access and retry if Hugging Face or OpenSLR is temporarily slow.
